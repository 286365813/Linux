# 内存 free

```yaml

[root@localhost ~]# free -h
              total        used        free      shared  buff/cache   available
Mem:           1.8G        431M        1.2G        9.5M        173M        1.2G
Swap:          2.0G          0B        2.0G
```

核心关系：

```yaml
total = used + free + buff/cache
```

## **buff/cache 的关系：**

Buffer 的话就是写入磁盘， Cache 的话就是文件从磁盘读出。

1.  **Buffer：**

```yaml
作用：写磁盘缓存
数据要写到硬盘，先放 Buffer，攒一批再一次性写入
主要是：磁盘读写、日志、文件写入
Java 打大量日志，Buffer 会涨
```

2.  **Cache**

```yaml
作用：读磁盘缓存
系统把常用文件、程序、jar 包、图片读到内存里，下次直接用，不用再读硬盘
Java 加载类、读取文件、读取数据库数据，Cache 会涨
```

## 重点关注核心：

3 个核心指标：

### 第一重要 `available` (可用内存)

| available | 状态 |
| --- | --- |
| >30% | 很健康 |
| 10%~30% | 正常 |
| <10% | 开始危险 |
| <5% | 容易 Full GC/OOM |
| 接近0 | 系统快炸 |

### 第二重要：Swap

```yaml
持续变少
说明：

已经开始大量换页
JVM 可能卡顿
GC 变慢
STW 变长

Java 最怕 swap。
```

# top 命令

```yaml
PID USER PR NI VIRT RES SHR S %CPU %MEM TIME+ COMMAND
| 字段      | 含义      | 重点程度  |
| ------- | ------- | ----- |
| PID     | 进程ID    | ⭐     |
| USER    | 进程所属用户  | ⭐     |
| PR      | 进程优先级   | ⭐⭐    |
| NI      | nice值   | ⭐⭐    |
| VIRT    | 虚拟内存    | ⭐⭐⭐   |
| RES     | 实际物理内存  | ⭐⭐⭐⭐⭐ |
| SHR     | 共享内存    | ⭐⭐    |
| S       | 进程状态    | ⭐⭐⭐   |
| %CPU    | CPU使用率  | ⭐⭐⭐⭐⭐ |
| %MEM    | 内存占比    | ⭐⭐⭐⭐⭐ |
| TIME+   | 累计CPU时间 | ⭐⭐⭐   |
| COMMAND | 进程名     | ⭐⭐⭐   |

```

# 三、内存占用过高 全套排查命令

1\. 找出占用内存最高的前 10 进程

```plain
ps -aux --sort=-%mem | head -10
```

2\. 看整机内存、剩余内存、缓存占用

```plain
free -h
```

3\. 按真实物理内存 RES 排序（最准，排除虚拟内存干扰）

```plain
ps -eo pid,user,rss,pcpu,pmem,comm --sort=-rss | head -10
```

rss 就是 top 里的 RES，**真实物理内存**

4\. 排查内存泄漏：持续观察内存变化

```plain
watch -n 1 free -h
```

## 5\. Java 内存高 / 泄漏排查命令

1.  查看堆内存使用概况

```plain
jstat -gc 进程PID 1000
```

1.  导出堆快照，分析泄漏

```plain
jmap -dump:format=b,file=heap.hprof 进程PID
```

6\. 查看系统是否发生 OOM 杀进程

```plain
dmesg | grep -i out
```
