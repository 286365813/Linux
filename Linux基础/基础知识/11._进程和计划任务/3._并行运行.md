在 Linux 中，并行执行指的是同时运行多个任务或命令，以提高效率，尤其是当任务可以独立执行时。常见的并行执行方法包括：

### 1\. **使用** `&` **将任务放入后台**

通过在命令末尾加上 `&`，可以将任务放到后台运行，多个后台任务可以同时执行。例如：

```bash
command1 &
command2 &
command3 &
```

这样，`command1`、`command2` 和 `command3` 会并行执行。

### 2\. **使用** `wait` **等待所有后台任务完成**

当你希望在脚本结束时等待所有后台任务完成，可以使用 `wait`：

```bash
command1 &
command2 &
command3 &
wait
```

`wait` 会等到所有后台任务执行完毕后再继续执行脚本。

### 3\. **使用** `xargs` **进行并行处理**

`xargs` 可以用来并行执行多个任务。`-P` 参数可以指定并行执行的任务数。例如：

```bash
echo -e "task1\ntask2\ntask3" | xargs -P 3 -I {} bash -c "echo {}; sleep 2"
```

这里，`xargs` 会同时执行三个任务，且每个任务都会输出任务名并延迟 2 秒。

### 4\. **在脚本中并行执行多个任务**

在脚本中，你可以将多个任务放在后台执行，并在最后使用 `wait` 等待它们完成。例如：

```bash
#!/bin/bash

# 启动多个任务并行执行
command1 &
command2 &
command3 &

# 等待所有后台任务完成
wait
```

### 6\. **使用** `&` **和** `wait` **的组合**

这是一个非常常见的在脚本中并行执行多个命令的方式：

```bash
#!/bin/bash

# 执行任务并行
task1 &
task2 &
task3 &

# 等待所有任务执行完
wait
echo "All tasks are done!"
```

脚本

```bash
NET="10.0.0"
for i in {1..254}; do
    {
        # 检查主机是否在线
        if ping -c1 -W1 ${NET}.${i} &> /dev/null; then
            # 如果在线，输出主机地址并追加到日志
            echo "${NET}.${i} is up" | tee -a hostlist.log
        else
            # 如果离线，输出主机地址
            echo "${NET}.${i} is down"
        fi
    } &
done
wait
```
