## 用户计划任务

```bash
[root@localhost ~]# cat /etc/crontab
SHELL=/bin/bash
PATH=/sbin:/bin:/usr/sbin:/usr/bin
MAILTO=root

# For details see man 4 crontabs

# Example of job definition:
# .---------------- minute (0 - 59)
# |  .------------- hour (0 - 23)
# |  |  .---------- day of month (1 - 31)
# |  |  |  .------- month (1 - 12) OR jan,feb,mar,apr ...
# |  |  |  |  .---- day of week (0 - 6) (Sunday=0 or 7) OR sun,mon,tue,wed,thu,fri,sat
# |  |  |  |  |
# *  *  *  *  * user-name  command to be executed

```

| **字段** | **说明** | **取值范围** |
| --- | --- | --- |
| **SHELL** | 指定执行任务时使用的 Shell 环境 | 例如：`/bin/bash` |
| **PATH** | 指定执行任务时的环境变量 `PATH`，即系统查找可执行文件的路径 | 例如：`/sbin:/bin:/usr/sbin:/usr/bin` |
| **MAILTO** | 指定任务执行结果的邮件接收者 | 例如：`root` |
| **分钟** | 任务执行的分钟 | 0 - 59 |
| **小时** | 任务执行的小时 | 0 - 23 |
| **日期** | 任务执行的日期（一个月中的哪一天） | 1 - 31 |
| **月份** | 任务执行的月份 | 1 - 12 或 jan,feb,mar,... |
| **星期** | 任务执行的星期（0 或 7 代表星期日） | 0 - 6 或 sun,mon,tue,... |
| **user-name** | 执行任务的用户（通常可以省略，默认为当前用户） | 例如：`root` |
| **command** | 需要执行的命令或脚本 | 例如：`/path/to/script.sh` |

**下面重点定义了时间用法；重点看一下**

**分、时、日、月、周：** **分钟、小时、30天的那天、第几个月、星期几。**

1.  在时间定义段中，使用`*`表示每单位，即每分钟，每小时，每天，每月，每周几(仍然是每天)。
2.  每个时间段中，都可以使用逗号`,`来表示枚举，例如定义`0,30,50 \* \* \* \*`表示每个时辰的整点、第30分钟和第50分钟都执行该任务。
3.  每个时间段中，都可以使用`-`定义范围，可以结合逗号使用。如分钟段定义了`00,20-30,50`表示每个时辰的整点、第20到30分钟的每分钟、第50分钟都执行该任务。
4.  在 cron 的时间设置字段中，`**/**` 用于表示“每”或“步长”。具体来说，`/` 后面的数字表示任务应该在特定单位的时间间隔内执行。 如在小时段定义了`0-13/2`表示在`0/2/4/6/8/10/12`点才满足时间定义。
5.  如果定义的日和周冲突了，则会多次执行(不包括因为`*`号导致的冲突)。例如**每月的15号执行该任务，同时又定义了周三执行该任务，正常无冲突情况下，将在周三和每月15号执行，但如果某月的15号同时是周三，**则该任务在此日执行两次。因此，应该尽力避免同时定义周和日的任务。
6.  另外一个需要注意的时间段设置是，使用\*号问题。例如`* */2 * * *`，它表示每隔两小时后的每一分钟都执行任务，也就是凌晨0点的每分钟执行任务，凌晨1点不执行任务，凌晨2点的每分钟执行任务，凌晨4点的每分钟执行任务，依此类推。同理，`*/5 */2 * * *`表示每隔2小时后的每5分钟执行一次任务。
7.  cron任务中不建议使用%，它有特殊用途，它表示换行的特殊意义，且第一个%后的所有字符串会被将成当作命令的标准输入，如果在命令中要使用%，则需要用 \\ 转义

```bash
30 2 * * * /bin/cp -a /etc/ /data/etc`date +\%F_\%T`
30 2 * * * /bin/cp -a /etc/ /data/etc`date +‘%F_%T’`   这个会有问题
```

示例任务：

| **示例** | **说明** |
| --- | --- |
| `* * * * * root /path/to/script.sh` | 每分钟以 `root` 用户身份执行 `/path/to/script.sh` |
| `0 3 * * * root /path/to/backup.sh` | 每天凌晨 3 点以 `root` 用户身份执行 `/path/to/backup.sh` |
| `0 0 1 * * root /path/to/cleanup.sh` | 每月 1 日的午夜 0 点以 `root` 用户身份执行 `/path/to/cleanup.sh` |
| `0 */2 * * * root /path/to/monitor.sh` | 每 2 小时以 `root` 用户身份执行 `/path/to/monitor.sh` |
| `30 8 * * 1-5 root /path/to/report.sh` | 每周一到周五的 8:30 以 `root` 用户身份执行 `/path/to/report.sh` |

## **命令用法：**

以下是将 `crontab --help` 的输出内容整理成的表格：

| **选项** | **说明** |
| --- | --- |
| **-u** | 指定操作用户的 crontab 文件（需要管理员权限） |
| **-e** | 编辑当前用户的 crontab 文件 |
| **-l** | 列出当前用户的 crontab 文件内容 |
| **-r** | 删除当前用户的 crontab 文件 |
| **-i** | 在删除 crontab 文件前提示确认 |
| **-n** | 在集群环境中，指定运行用户 crontab 的主机 |
| **-c** | 在集群环境中，获取运行用户 crontab 的主机 |
| **-s** | 设置 SELinux 安全上下文 |
| **-x** | 启用调试模式，`<mask>` 是调试级别 |

使用示例：

| **命令** | **说明** |
| --- | --- |
| `crontab -e` | 编辑当前用户的 crontab 文件 |
| `crontab -l` | 列出当前用户的 crontab 文件内容 |
| `crontab -r` | 删除当前用户的 crontab 文件 |
| `crontab -i -r` | 删除当前用户的 crontab 文件前提示确认 |
| `crontab -u root -e` | 编辑 `root` 用户的 crontab 文件（需要管理员权限） |
| `crontab -n hostname -e` | 在集群环境中，编辑指定主机的 crontab 文件 |
| `crontab -x 1` | 启用调试模式，调试级别为 1 |

2.  编写磁盘脚本

```bash

[root@localhost ~]# cat disk_check.sh
#!/bin/bash

WARNING=10

# 获取磁盘使用情况，并检查是否超过警告值
df | sed -En '/^\/dev\/sd/s@^([^ ]+).* ([0-9]+)%.*@\1 \2@p' | while read DEVICE USE
do
    # 如果使用率超过警告值，发送邮件警告
    if [ $USE -gt $WARNING ]; then
        echo "$DEVICE will be full, USE: $USE%" | mail -s "Disk Full Warning" 710914818@qq.com
    fi
done
```

3.  设置定时任务

```bash

[root@localhost ~]# crontab -e
*/10 8-18 * * 0-6 /root/disk_check.sh

```

# 定时任务的环境变量

先看看定时任务执行的环境变量，从下面例子可以看到当前变量是无法执行命令的（对应命令存放 ： /usr/sbin）；我们要在计划任务中指定环境变量三种方法 ： 计**划任务中指定 、 计划任务使用绝对命令 、 计划任务使用脚本，脚本中指定变量。**

```bash

[root@localhost ~]# crontab -l
* * * * *  echo $PATH > /root/path.txt

[root@localhost ~]# cat path.txt
/usr/bin:/bin

[root@localhost ~]# echo $PATH
/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/root/bin

```

1.  计划任务指定

```bash

[root@localhost ~]# crontab -l
PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/root/bin
* * * * *  echo $PATH > /root/path.txt

[root@localhost ~]# cat path.txt
/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/root/bin
```

2.  使用绝对路径

```bash

[root@localhost ~]# crontab -l
* * * * *  /usr/sbin/useradd user$RANDOM
```

3.  使用脚本

```bash

[root@localhost ~]# cat adduser.sh
#!/bin/bash
export PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/root/bin
useradd "user$RANDOM"

[root@localhost ~]# chmod +x adduser.sh

[root@localhost ~]# crontab -l
* * * * *  /root/adduser.sh
```
