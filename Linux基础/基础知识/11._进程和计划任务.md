## 什么是进程

Process（进程）: 从硬盘读取程序的副本；被载入内存的一个指令集合，是资源分配的单位

-   进程ID（PID）号码被用来标记各个进程
-   UID、GID、和SELinux语境决定对文件系统的存取和访问权限
-   通常从执行进程的用户来继承
-   存在生命周期

进程创建

-   init：第一个进程，从 CentOS7 以后为systemd
-   进程：都由其父进程创建，fork()，父子关系。

**进程和线程由系统调度；** **协程是由程序员显式地管理的轻量级线程**

## 进程状态

![image.png](assets/11._进程和计划任务-4.png)

-   创建状态：进程在创建时需要申请一个空白PCB(process control block进程控制块)，向其中填写控制和管理进程的信息，完成资源分配。如果创建工作无法完成，比如资源无法满足，就无法被调度运行，把此时进程所处状态称为创建状态
-   就绪状态：进程已准备好，已分配到所需资源，只要分配到CPU就能够立即运行
-   执行状态：进程处于就绪状态被调度后，进程进入执行状态
-   阻塞状态：正在执行的进程由于某些事件（I/O请求，申请缓存区失败）而暂时无法运行，进程受 到阻塞。在满足请求时进入就绪状态等待系统调用
-   终止状态：进程结束，或出现错误，或被系统终止，进入终止状态。无法再执行

**状态之间转换六种情况**

-   **运行——>就绪** 1，主要是进程占用CPU的时间过长，而系统分配给该进程占用CPU的时间是有限的； 2，在采用抢先式优先级调度算法的系统中,当有更高优先级的进程要运行时，该进程就被迫让出CPU， 该进程便由执行状态转变为就绪状态 。
-   **就绪——>运行：**运行的进程的时间片用完，调度就转到就绪队列中选择合适的进程分配CPU 。
-   **运行——>阻塞：**正在执行的进程因发生某等待事件而无法执行，则进程由执行状态变为阻塞状态，如 发生了I/O请求 。
-   **阻塞——>就绪:**进程所等待的事件已经发生，就进入就绪队列

**以下两种状态是不可能发生的：**

-   阻塞——>运行：即使给阻塞进程分配CPU，也无法执行，操作系统在进行调度时不会从阻塞队列进行 挑选，而是从就绪队列中选取
-   就绪——>阻塞：就绪态根本就没有执行，谈不上进入阻塞态

**进程更多的状态：**

-   **运行态：running**
-   **就绪态：ready**
-   **睡眠态：分为两种，可中断：interruptable，不可中断：uninterruptable**
-   **停止态：stopped，暂停于内存，但不会被调度，除非手动启动**
-   **僵死态：zombie，僵尸态，结束进程，父进程结束前，子进程不关闭，杀死父进程可以关闭僵死 态 的子进程**

## **进程信息 ps**

命令格式：

1.  **UNIX 风格选项**

-   \-A 或 \-e：显示所有进程。
-   \-a：显示所有用户的进程，但不包括会话领导和没有控制终端的进程。
-   \-u：显示进程的用户格式。
-   \-f：显示完整格式的输出。
-   \-l：显示长格式。
-   \-p：通过进程 ID 过滤。
-   \-t：通过终端过滤。

```bash
ps -e
ps -ef
ps -u username
ps -p 1234
ps -t pts/0
```

2.  **BSD 风格选项**

-   a：显示所有用户的进程，包括那些没有控制终端的进程。
-   x：显示没有控制终端的进程。
-   u：以用户格式显示进程。
-   r：仅显示正在运行的进程。
-   o：指定输出的格式。 pid,ppid,user,%cpu,%mem,comm

```bash
ps aux
ps ax
ps au
```

3.  **GNU 长选项风格**

-   \--help：显示帮助信息。
-   \--version：显示版本信息。
-   \--forest：以树状结构显示进程。
-   \--sort：指定排序字段。
-   \--pid：通过进程 ID 过滤。
-   \--ppid：通过父进程 ID 过滤。

```bash
ps --help
ps --version
ps --forest
ps --pid 1234
ps --sort=-%mem

ps axo pid,cmd,%cpu,%mem k -%cpu   按CPU利用率倒序排序
ps axo pid,cmd,%cpu,%mem --sort -%mem  按内存倒序排序
```

4.  **BSD 风格输出说明**

`**ps aux**`**输出说明：**

```bash
USER       PID %CPU %MEM    VSZ   RSS TTY      STAT START   TIME COMMAND
root         1  0.0  0.1  19368  1348 ?        Ss   Mar30   0:01 /usr/lib/systemd/systemd --switched-root --system --deserialize 21
root         2  0.0  0.0      0     0 ?        S    Mar30   0:00 [kthreadd]
```

字段说明

-   **USER**：进程的所有者（用户）。

-   例如，root 表示这个进程是由 root 用户启动的。

-   **PID**：进程 ID。

-   例如，1 是系统初始化进程的 PID。

-   **%CPU**：进程使用的 CPU 时间的百分比。

-   例如，0.0 表示几乎没有使用 CPU。

-   **%MEM**：进程使用的物理内存的百分比。

-   例如，0.1 表示使用了系统总物理内存的 0.1%。

-   **VSZ**：虚拟内存集的大小（以 KB 为单位）。

-   例如，19368 表示虚拟内存大小为 19368 KB。

-   **RSS**：驻留集大小（以 KB 为单位），即进程实际使用的物理内存。

-   例如，1348 表示驻留内存大小为 1348 KB。

-   **TTY**：进程关联的终端。

-   例如，? 表示进程没有控制终端。

-   **STAT**：进程状态码。常见的状态码包括：

-   R：运行（Running）
-   S：休眠（Sleeping）
-   D：不可中断的睡眠（Uninterruptible sleep）
-   T：停止（Stopped）
-   Z：僵尸进程（Zombie）
-   Ss：主进程，处于睡眠状态
-   R+：在前台运行的进程

-   **START**：进程启动的时间或日期。

-   例如，Mar30 表示该进程在 3 月 30 日启动。

-   **TIME**：进程使用的总 CPU 时间。

-   例如，0:01 表示使用了 1 秒的 CPU 时间。

-   **COMMAND**：启动进程的命令及其参数。

-   例如，/usr/lib/systemd/systemd --switched-root --system --deserialize 21 是启动该进程的完整命令。

5.  **UNIX 风格选项 输出说明**

`ps -ef`输出说明：

```bash
UID        PID  PPID  C STIME TTY          TIME CMD
root         1     0  0 08:01 ?        00:00:01 /usr/lib/systemd/systemd --switched-root --system --deserialize 21
root         2     0  0 08:01 ?        00:00:00 [kthreadd]

```

-   **UID**：进程的所有者（用户）。

-   例如，root 表示该进程由 root 用户启动。

-   **PID**：进程 ID。

-   例如，1 是系统初始化进程的 PID。

-   **PPID**：父进程 ID。

-   例如，0 表示没有父进程（即系统初始化进程），2 表示父进程是 PID 为 2 的进程。

-   **C**：CPU 使用率，表示该进程使用的 CPU 时间的百分比。

-   例如，0 表示该进程几乎没有使用 CPU。

-   **STIME**：进程启动的时间或日期。

-   例如，08:01 表示该进程在当天的 08:01 启动。

-   **TTY**：终端类型。

-   例如，? 表示该进程没有控制终端。

-   **TIME**：进程使用的总 CPU 时间。

-   例如，00:00:01 表示使用了 1 秒的 CPU 时间。

-   **CMD**：启动进程的命令及其参数。

-   例如，/usr/lib/systemd/systemd --switched-root --system --deserialize 21 是启动该进程的完整命令。

### 查找进程 **pgrep**

命令格式：`pgrep [options] pattern`

```bash
-u 实际用户ID (Real User ID, ruid)
-U 有效用户ID (Effective User ID, euid)
-t terminal: 与指定终端相关的进程
-l: 显示进程名
-a: 显示完整格式的进程名
-P pid: 显示指定进程的子进程

u：ruid (Real User ID, ruid) 和 -U (Effective User ID, euid)区别。
● -u 选项：匹配进程的实际用户ID。
● 实际用户ID指的是启动进程的用户ID，也就是拥有该进程的用户。
● 即使进程以不同的权限运行，实际用户ID不会改变。

有效用户ID (Effective User ID, euid)
●-U 选项：匹配进程的有效用户ID。
● 有效用户ID指的是进程在运行时使用的用户ID。
● 进程可以通过某些机制（例如suid）获得与启动它的用户不同的权限，因此有效用户ID可能与实际用户ID不同。
```

# **系统负载性能相关工具**

Load Average的三个数字，依次则是过去1分钟、5分钟、15分钟的平均负载。可以通过观察这三个数字的大小，可以简单判断系统的负载是下降的趋势还是上升的趋势。**负载值一般不超过cpu核数的1-1.5倍，如果超过1.5倍，那就要重视，此时会严重影响系统。**

-   如果 load average: 1.00, 5.00, 10.00 三个数字依次增大，则说明在过去的 1 分钟系统的负载比过去 15 分钟系统的负载小，表明系统的负载是下降的趋势。
-   如果 load average: 10.00, 5.00, 1.00 三个数字依次降低，则说明在过去的 1 分钟系统的负载比过去 15 分钟系统的负载大，表明系统的负载是上升的趋势。
-   如果 load average: 0.07, 0.04, 0.0 三个数字基本相同，或者相差不大， 表明系统的负载是平稳的。

## **负载查询 uptime**

```bash
[litao@localhost ~]$ uptime
 22:17:26 up  8:20,  1 user,  load average: 0.00, 0.01, 0.05
 
1. 系统时间-UP状态-开机时间-用户-1分钟、5分钟、15分钟系统负载。

2. Load average的值代表了系统中正在运行或等待运行的进程数量。负载平均值高表示系统上有更多的工作需要处理。一般来说，如果负载平均值超过CPU核心数量的70%~80%，就可以认为系统负载较高。

3. 如果你的系统只有一颗CPU，而每颗CPU有两个核心，那么你的系统总共有两个物理核心。一般来说，如果负载平均值超过1.4~1.6，即超过核心数的70%~80%，就可以认为系统负载较高。

```

-   us：用户态CPU使用率。
-   sy：内核态CPU使用率。
-   ni：低优先级用户态CPU使用率。
-   id：空闲CPU比例。
-   wa：等待I/O的CPU时间比例。
-   hi：硬件中断使用的CPU时间比例。
-   si：软件中断使用的CPU时间比例。
-   st：虚拟机窃取的CPU时间比例。

| `mode` | 含义 | 说明 |
| --- | --- | --- |
| **user** | 用户态 | CPU 在执行用户进程（不含 nice 进程） |
| **nice** | 优先级调整过的用户进程 | 通常是低优先级的用户进程 |
| **system** | 内核态 | CPU 在执行内核代码（系统调用等） |
| **idle** | 空闲状态 | 没有任务执行，CPU在“休息” |
| **iowait** | 等待 IO | CPU 等磁盘或网络 IO 完成 |
| **irq** | 硬中断 | 处理中断请求（硬件） |
| **softirq** | 软中断 | 内核中处理任务较轻的中断 |
| **steal** | 被偷时间 | 虚拟机中，CPU 被其他虚机“抢走”了 |
| **guest** | 来宾态 | 虚拟机运行的时间（不常见） |

-   total：总交换分区大小。

5.  进程信息

```bash
  PID USER      PR  NI    VIRT    RES    SHR S  %CPU %MEM     TIME+ COMMAND
```

-   PID：进程ID。
-   USER：进程所有者。
-   PR：优先级。
-   NI：nice值（影响进程优先级）。
-   VIRT：进程使用的虚拟内存总量。
-   RES：进程使用的驻留内存量。
-   SHR：进程使用的共享内存量。
-   S：进程状态（S：休眠，R：运行，T：停止，Z：僵尸）。
-   %CPU：进程的CPU使用率。
-   %MEM：进程的内存使用率。
-   TIME+：进程启动以来的总CPU时间。
-   COMMAND：启动进程的命令名。

### **top的内置命令**

1.  排序：

## 查看内存 free

1.  命令格式： `free [OPTION]`

-   \-m：以MB为单位显示内存信息。

2.  内容输出

```bash
              total        used        free      shared  buff/cache   available
Mem:        16347908     6138236     2386344      450364     7833328     9213072
Swap:       2097148           0     2097148
```

**Mem**行：

-   total：16347908 KiB（总内存量）
-   used：6138236 KiB（已使用的内存量）
-   free：2386344 KiB（空闲的内存量）
-   shared：450364 KiB（与tmpfs共享的内存量）
-   buff/cache：7833328 KiB（用于缓存和缓冲的内存量）
-   available：9213072 KiB（可用的内存量）

**Swap**行：

-   total：2097148 KiB（总交换分区量）
-   used：0 KiB（已使用的交换分区量）
-   free：2097148 KiB（空闲的交换分区量）

3.  字段意思

-   total = used + free + buff/cache
-   used = total - free - buff/cache
-   available ≈ free + reclaimable part of buff/cache + some active/inactive memory

## **虚拟内存信息 vmstat**

用于报告系统的虚拟内存、进程、CPU活动等的统计信息。注意；这里的si和so；bi和bo是以内存为参照。

1.  **procs**

```bash
 2  1 466184  52124      0  41740  116 530292   804 530292 29769 2100  0 82 17  2  0
 0  0  48128 1686440      0  41432  860  172 12984   204 1127  723  0 15 85  1  0

```

**交换到内存（Swap-in）**

-   si 值从 116 增至 860：

-   这表示在执行 dd 命令时，每秒有 116 KB 数据从磁盘交换到内存。命令执行完成后，这一速率增加到 860 KB/s，表示系统在命令执行期间需要频繁从交换空间读取数据以满足内存需求。

**交换到磁盘（Swap-out）**

-   so 值从 530292 降至 172：

-   这表示在执行 dd 命令时，每秒有 530292 KB 数据从内存交换到磁盘。这个值非常高，表明在命令执行期间，系统将大量数据交换到磁盘，以腾出内存来处理当前操作。
-   命令执行完成后，交换到磁盘的速率恢复正常，降至 172 KB/s，这说明交换操作主要在 dd 命令执行期间发生。

**bi 和 bo 的高值：**

-   bi = 804：

-   这表示每秒从块设备读取 804 块数据。

-   bo = 530292：

-   这表示每秒向块设备写入 530292 块数据。这个值非常高，表明在 dd 命令执行期间，有大量数据写入到块设备。

**so = 530292（交换到磁盘的速率）：与 bo 值相等。**

-   这表明在执行 dd 命令期间，系统确实将大量数据交换到磁盘，这导致了高 bo 值。交换操作的增加是由于内存压力或其他资源限制引起的。

## **mpstat-统计CPU性能和历史性能数据**

显示每个CPU的使用情况：`mpstat -P ALL`

输出：

```bash
Linux 5.4.0-54-generic (hostname) 	06/04/2024 	_x86_64_	(4 CPU)

09:30:01 AM  CPU    %usr   %nice    %sys %iowait    %irq   %soft  %steal   %idle

```

**1CPU**：显示统计信息的 CPU 核心编号。all 表示所有 CPU 的平均统计信息。

-   **%usr**（User）：在用户级别（应用程序）运行的时间百分比，不包括 nice 值调整的进程。

-   例如，%usr 为 2.50 表示 CPU 有 2.50% 的时间用于执行用户空间的进程。

-   **%nice**（Nice）：在用户级别运行的时间百分比，包括那些已调整 nice 值的进程。

-   例如，%nice 为 0.00 表示没有 CPU 时间用于执行调整了 nice 值的进程。

-   **%sys**（System）：在内核级别（系统）运行的时间百分比。

-   例如，%sys 为 1.25 表示 CPU 有 1.25% 的时间用于执行内核代码。

-   **%iowait**（I/O Wait）：CPU 等待 I/O 操作完成的时间百分比。

-   例如，%iowait 为 0.75 表示 CPU 有 0.75% 的时间在等待 I/O 操作。

-   **%irq**（IRQ）：处理硬中断的时间百分比。

-   例如，%irq 为 0.00 表示没有 CPU 时间用于处理硬中断。

-   **%soft**（Soft IRQ）：处理软中断的时间百分比。

-   例如，%soft 为 0.25 表示 CPU 有 0.25% 的时间用于处理软中断。

-   **%steal**（Steal）：虚拟化环境中被其他虚拟机窃取的时间百分比。

-   例如，%steal 为 0.00 表示没有 CPU 时间被其他虚拟机窃取。

-   **%guest**（Guest）：客户操作系统运行的时间百分比（虚拟化）。

-   例如，%guest 为 0.00 表示没有 CPU 时间用于执行虚拟机客户代码。

-   **%gnice**（Guest Nice）：客户操作系统的 nice 值调整的时间百分比（虚拟化）。

-   例如，%gnice 为 0.00 表示没有 CPU 时间用于执行调整了 nice 值的虚拟机客户代码。

-   **%idle**（Idle）：CPU 空闲时间百分比。

-   例如，%idle 为 95.25 表示 CPU 有 95.25% 的时间是空闲的。
