# sudo授权规则配置

配置文件路径： /etc/sudoers, /etc/sudoers.d/ 和授权规则

```bash
user host=(run-as-user:run-as-group) command
```
> -   **user**: 允许执行命令的用户名。
> -   **host**: 允许执行命令的主机，可以是主机名、IP地址，或者 **ALL** 表示所有主机。
> -   **run-as-user**: 以哪个用户的身份运行命令。
> -   **run-as-group**: 以哪个组的身份运行命令。
> -   **command**: 允许执行的命令，可以是具体的命令，也可以使用通配符。

**例如：**

```bash
john ALL=(ALL:ALL) NOPASSWD: /usr/bin/apt-get
表示用户 John 可以在任何主机上以任何用户和组的身份无需密码运行 /usr/bin/apt-get。

dwlitao7 ALL=(ALL) NOPASSWD:ALL
表示用户 dwlitao7 在所有主机上（ALL）可以作为任何用户（ALL）运行sudo命令，NOPASSWD:ALL: 表示用户 dwlitao7 在执行sudo命令时不需要输入密码。
```

**特殊标记：****/etc/sudoers** 中有一些特殊标记，如 **ALL** 和 **NOPASSWD**。

> -   **ALL**：表示适用于所有用户、主机、命令或组。
> -   **NOPASSWD**：表示无需输入密码即可执行相应的命令。

**组规则：** 除了用户之外，还可以使用 **%** 符号指定组。例如：

```bash
%admins ALL=(ALL:ALL) ALL
```

表示组 **admins** 中的所有用户可以在任何主机上以任何用户和组的身份运行任何命令

**命令别名：** 为了简化规则，可以使用 **Cmnd\_Alias** 定义命令别名。例如：

```bash
Cmnd_Alias APT = /usr/bin/apt-get, /usr/bin/apt-cache
john ALL=(ALL:ALL) APT
```

## sudo配置别名

1.  **User\_Alias（用户别名）**：将一组用户合并到一个别名中，以便在**sudoers**文件中更轻松地引用。例如：

```bash
User_Alias ADMINS = user1, user2, user3
```

2.  **Runas\_Alias（目标用户别名）**：将一组目标用户合并到一个别名中，以便在**sudoers**文件中更轻松地引用。这用于指定在执行命令时可以切换到的用户。例如：

```bash
Runas_Alias SYSADMINS = root, sysadmin
```

3.  **Host\_Alias（主机别名）**：将一组主机合并到一个别名中，以便在**sudoers**文件中更轻松地引用。这用于指定在哪些主机上可以执行**sudo**命令。例如：

```bash
Host_Alias SERVERS = server1, server2
```

4.  **Cmnd\_Alias（命令别名）**：将一组命令合并到一个别名中，以便在**sudoers**文件中更轻松地引用。这用于指定哪些命令可以由用户以特定的身份执行。例如：

```bash
Cmnd_Alias SHUTDOWN_CMNDS = /sbin/shutdown, /sbin/reboot
```
