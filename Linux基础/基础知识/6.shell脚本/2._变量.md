# **命名要求**

-   区分大小写
-   不能使程序中的保留字和内置变量：如：if, for
-   只能使用数字、字母及下划线，且不能以数字开头，注意：不支持短横线 “ - ”，和主机名相反

# **变量定义和引用**

-   普通变量：生效范围为当前shell进程；对当前shell之外的其它shell进程，包括当前shell的子shell 进程均无效

-   环境变量：生效范围为当前shell进程及其子进程
-   本地变量：生效范围为当前shell进程中某代码片断，通常指函数

# 变量

## 3.1 变量赋值

```plsql
直接字串：name='root'
变量引用：name="$USER"
命令引用：name=`COMMAND` 或者 name=$(COMMAND)

弱引用和强引用
"$name" 弱引用，其中的变量引用会被替换为变量值
'$name' 强引用，其中的变量引用不会被替换为变量值，而保持原字符串
```
```plsql
直接字符串：
[root@litao ~]# NAME='litao'
[root@litao ~]# echo $NAME
litao

变量引用：
[root@litao ~]# TITLE=cto
[root@litao ~]# echo $TITLE
cto
[root@litao ~]# NAME=wang
[root@litao ~]# TITLE=$NAME
[root@litao ~]# echo $TITLE
wang
[root@litao ~]# echo $NAME
wang

命令引用：name=`COMMAND` 或者 name=$(COMMAND)
[root@litao ~]# IP=`echo 192.168.1.{1..10}`
[root@litao ~]# echo $IP
192.168.1.1 192.168.1.2 192.168.1.3 192.168.1.4 192.168.1.5 192.168.1.6 192.168.1.7 192.168.1.8 192.168.1.9 192.168.1.10

变量追加；
[root@litao ~]# TITLE=CTO
[root@litao ~]# echo $TITLE
CTO
[root@litao ~]# TITLE+=:li  //变量追加+=
[root@litao ~]# echo $TITLE
CTO:li

查看当前变量和取消变量
set 
[root@litao ~]# echo $TITLE
CTO:li
[root@litao ~]# unset TITLE
[root@litao ~]# echo $TITLE

[root@litao ~]# 

```

## 3.2 环境变量

### 3.2.1普通变量

普通变量子进程无法继承父进程变量

```plsql
[root@litao ~]# pstree -p | grep sshd
           |-sshd(877)---sshd(1597)---bash(1599)-+-grep(1639)
[root@litao ~]# echo $BASHPID
1599
[root@litao ~]# CEO='litao'
[root@litao ~]# echo $CEO
litao
[root@litao ~]# bash
[root@litao ~]# echo $BASHPID
1641
[root@litao ~]# echo $CEO   //返回空值

[root@litao ~]# echo $PPID
1599
[root@litao ~]# exit
exit
[root@litao ~]# echo $CEO
litao
```

### 3.2.2 环境变量

环境变量声明和赋值：

-   可以使子进程（包括孙子进程）继承父进程的变量，但是无法让父进程使用子进程的变量
-   一旦子进程修改从父进程继承的变量，将会新的值传递给孙子进程

```plsql
[root@litao ~]# export age=18   //定义环境变量
[root@litao ~]# echo $BASHPID
1659
[root@litao ~]# bash
[root@litao ~]# echo $BASHPID
1677
[root@litao ~]# echo $PPID
1659
[root@litao ~]# echo $age
18
## 再次生成子shell还能看到定义的环境变量
[root@litao ~]# bash
[root@litao ~]# pstree -p | grep sshd
           |-sshd(877)---sshd(1657)---bash(1659)---bash(1677)---bash(1688)-+-grep(1700)
[root@litao ~]# echo $BASHPID
1688
[root@litao ~]# echo $age
18

## 修改父进程继承的变量，父进程变量不会变，新的值会给孙子进程。
[root@litao ~]# age=19
[root@litao ~]# echo $age
19
[root@litao ~]# exit
exit
[root@litao ~]# echo $age
18

再生成子shell，会继承上级的变量
[root@litao ~]# bash 
[root@litao ~]# echo $age
19

环境变量查看 env export
[root@litao ~]# export
declare -x EDITOR="vim"
...
declare -x age="18"

```

### 3.2.3 只读变量

只读变量：只能声明定义，但后续不能修改和删除，即常量

```plsql
[root@litao ~]# readonly PI=3.14159  //定义只读变量
[root@litao ~]# echo $PI
3.14159
[root@litao ~]# bash    //子进程无法继承只读变量
[root@litao ~]# echo $PI

[root@litao ~]# exit
exit
[root@litao ~]# echo $PI
3.14159
[root@litao ~]# PI='999'   //提示PI是个只读变量，无法修改
bash: PI: readonly variable

查看只读变量
[root@litao ~]# declare -r
...
declare -r PI="3.14159"
declare -ir PPID="1704"

只读变量无法删除，只能退出当前shell
[root@litao ~]# unset PI
bash: unset: PI: cannot unset: readonly variable
```

### 3.2.4 位置变量

-   $1, $2, ... 对应第1个、第2个等参数，shift [n]换位置 、$0命令本身,包括路径
-   $\*传递给脚本的所有参数，全部参数合为一个字符串 、$@传递给脚本的所有参数，每个参数为独立字符串
-   $#传递给脚本的参数的个数
-   注意：$@ $\* 只在被双引号包起来的时候才会有差异

```plsql
[root@litao ~]# cat test.sh
#!/bin/bash
#
#**********************************************************************************************
#Author:        litao
#QQ:            286365813
#Date:          2023-10-08
#FileName:      test.sh
#URL:           www.baidu.com
#Description:   The test script
#Copyright (C): 2023 All rights reserved
#********************************************************************************************
echo "脚本名称是 $0"
echo "第1个变量是 $1"
echo "第2个变量是 $2"
echo "第3个变量是 $3"
echo "所有变量是 $@"
echo "所有变量是 $*"
echo "变量的个数有 $#"

[root@litao ~]# bash test.sh a b c d
脚本名称是 test.sh
第1个变量是 a
第2个变量是 b
第3个变量是 c
所有变量是 a b c d
所有变量是 a b c d
变量的个数有 4

$@认为 a b c d为单独的变量，如'a'、'b'、'c'、'd'
$*认为 'a b c d' 是一个变量
但是，$@ $* 只在被双引号包起来的时候才会有差异
```

## 退出状态码变量

$?的值为0 #代表成功

$?的值是1到255 #代表失败

```plsql
[root@litao ~]# ping www.baidu.com
PING www.a.shifen.com (14.119.104.254) 56(84) bytes of data.
64 bytes from 14.119.104.254 (14.119.104.254): icmp_seq=1 ttl=128 time=8.36 ms
...
rtt min/avg/max/mdev = 8.287/8.327/8.367/0.040 ms
[root@litao ~]# echo $?   
0
[root@litao ~]# xixi
bash: xixi: command not found
[root@litao ~]# echo $?
127
```

脚本中一旦遇到exit命令，脚本会立即终止；终止退出状态取决于exit命令后面的数字

```plsql
[root@litao ~]# cat test.sh
#!/bin/bash
echo "脚本名称是 $0"
exit
echo "第1个变量是 $1"
echo "第2个变量是 $2"
echo "第3个变量是 $3"
echo "所有变量是 $@"
echo "所有变量是 $*"
echo "变量的个数有 $#"

[root@litao ~]# bash test.sh 1 2 3
脚本名称是 test.sh

[root@litao ~]# echo $?
0
```

如果未给脚本指定退出状态码，整个脚本的退出状态码取决于脚本中执行的最后一条命令的状态码

```plsql
[root@litao ~]# bash test.sh 1 2 3
脚本名称是 test.sh
第1个变量是 1
第2个变量是 2
```
